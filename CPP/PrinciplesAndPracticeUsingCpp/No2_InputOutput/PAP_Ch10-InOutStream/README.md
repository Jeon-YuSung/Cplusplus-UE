## <개요>
10 ~ 11장은 C++ 표준 라이브러리에서 다양한 대상을 이용해 입출력을 제공하는 I/O 스트림에대해 알아본다. <br>
특히 파일을 읽고 쓰는 방법, 오류를 처리하는 방법, 형식화된 입력을 다루는 방법, 사용자 정의 타입에 I/O 연산자를 추가하고 활용하는 방법을 설명한다. <br>
10장은 개별적인 값을 읽고 쓰는 방법, 전체 파일을 열여서 읽고 쓰는 방법 등을 비롯한 기본적 입출력 모델에 대해 설명하고, 11장은 더 상세한 내용을 파고들것이다. 

### 10-1. 입력과 출력 
데이터가 없는 컴퓨팅은 무의미하다. 데이터를 프로그램 안으로 읽어야 하고, 계산 결과는 밖으로 출력해야 흥미로운 계산이 된다. <br> 
특수한 경우가 아닌 일반적인 용도의 프로그램이라면 프로그램이 데이터를 읽고 쓰는 방법과 실제 입출력 장치를 분리시켜야한다 <br>
현대 운영체제는 입출력 장치를 제어하는 일을 장치 드라이버에게 위임하고, 애플리케잇견은 서로 다른 장치의 입출력을 최대한 비슷하게 만들어주는 입출력 라이브러리를 바탕으로 장치 드라이버에 접근한다. 일반적으로 장치 드라이버는 운영체제의 깊숙한 곳에 위치 하기 때문에 대부분의 사용자가 볼 수 없고, 입출력 라이브러리가 입출력에 추상화를 제공하므로, 프로그래머가 장치나 장치 드라이버를 고려하지 않아도 된다. 

데이터 출처 : 입력 장치 → 장치 드라이버 → 입력 라이브러리 <br>
→ 프로그램 <br>
→ 데이터 목적지 : 출력 라이브러리 → 장치 드라이버 → 출력 장치 

입출력은 입출력 라이브러리가 제어하는 바이트(문자)의 스트림으로 볼 수 있다. <br> 
응용 프로그래머가 해야 할 일은 다음과 같다. <br>

1. 적절한 데이터 출처와 목적지에 입출력 스트림을 구성한다.
2. 이 스트림에 읽고 쓴다.

장치의 문자열 송수신에 대한 디테일은 입출력 라이브러리와 장치 드라이버가 담당하기에, 10 ~ 11장은 C++ 라이브러리를 사용해서 형식화된 데이타 스트림을 바탕으로 입출력 수행 하는 방법을 알아보는 챕터이다. <br>
프로그래머의 관점에서 다양한 종류의 입출력이 있는데, 다음과 같이 분류할 수 있다. <br>

1. (다수의) 데이터 항목의 스트림 (파일과 네트워크 연결, 리코딩 장치, 디스플레이 장치 등등...)
2. 키보드를 바탕으로 한 유저와의 의사소통 → CLI
3. 그래픽 인터페이스를 바탕으로 한 유저와의 의사소통 (시각적인 객체 출력, 마우스 클릭 읽기 등..) → GUI

이러한 분류는 유일하지도, 명확하지도 않은 분류 방법이다 <br>
예를 들어 문자열 출력 스트림의 목적지가 브라우저에서 사용할 HTTP 문서라면 그 출력 결과는 사용자 의사소통에 목적이 있고, 그래픽 요소를 포함할 수 도 있다. 반대로 GUI를 바탕으로 한 상호작용의 결과는 프로그램에게 문자열 스트림으로 제공될 수 있다. <br>
그러나, 이 책에서 사용할 도구에는 위와 같은 분류가 잘 맞는 편인데, 1번과 2번의 입출력 종류는 C++ 표준 라이브러리의 입출력 스트림과 대부분 현대 운영체제에서 직접적인 지원을 해주기 때문이다. 3번 같은 경우는 다양한 라이브러리에서 지원하지만, 그래픽스를 이용한 방법은 12 ~ 16장을 통해 알아본다. 

### 10-2. 입출력 스트림 모델
C++ 표준 라이브러리에서는 입력 스트림을 처리하는 istream 타입과 출력 스트림을 다루는 ostream 타입을 제공한다. <br>
std::cin은 표준 istream, std::cout은 표준 ostream을 사용한다. <br>

ostream의 특징
1. 다양한 타입의 값을 문자열로 변환한다.
2. 그 문자열을 어딘가(콘솔, 파일, 메인 메모리 등등)로 보낸다 

다양한 타입의 값 ('C', (12,34), 1232154 etc...) → ostream → **Buffer** → "어딘가"

버퍼는 ostream이 운영체제와 통신하는 동안 넘겨준 데이터를 저장하는 자료구조로, ostream에 데이터를 쓴 시점과 실제 목적지에 그 문자여일이 나타나는 시점 사이에 지연이 있다면, 문자열이 아직 버퍼에 있기 때문 가능성이 높다. 버퍼링은 성능에 큰 영향을 미치기 때문에, 성능이 중요한 대규모의 데이터를 다룰 때 중요하다. 

istream의 특징
1. 문자열을 여러 가지 타입의 값으로 변환한다.
2. 어딘가(콘솔, 파일, 메인 메모리 etc...)에서 그 문자열을 읽어온다.

 "어딘가" → Buffer → istream → 다양한 타입의 값   

istream도 운영체제와 통신할 때 버퍼를 사용하는데, istream의 버퍼는 사용자의 관점에서 더 쉽게 인식할 수 있다. 키보드에 연결된 istream을 사용할 때, 입력한 내용(리턴, 개행)은 키가 눌릴 때까지 버퍼에 남는다. 따라서, 엔터를 누르기전에 수정을 하고싶다면, 백스페이스 키를 통해 수정할 수 있다. <br>
출력의 주요 용도는 사람이 읽을 수 있는 데이터를 만드는데에 있다. 따라서, ostream은 다양한 취향에 맞게 텍스트를 형식화하는 기능을 제공하고, istream은 ostream에서 만들어진 그러한 종류의 출력을 다시 읽을 수 있는 기능을 제공한다.
