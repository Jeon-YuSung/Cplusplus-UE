## 5장 반복문

반복문에 대해 이해한다 <br>
while, for, do-while, 중첩 반복문에 대해 배운다 

-------------
### 5.1 반복문의 개요 

반복문의 필요한 이유 : 
+ 조건 분기를 사용하여 프로그램을 100번 실행할 수 있으나, 많은 시간이 소요됨<br>
+ 하나의 프로그램 내부에 조건 분기를 100번 사용하고 프로그램을 1회만 실행할 수 있음<br>
하지만 코드가 굉장히 길어지고, 조건이나 수가 바뀌면 프로그램을 다시 변경해야함<br>
+ 코드를 정해진 만큼 반복할수 있는 반복문(Repetition Statement)을 사용해서 프로그램을 작성할 수 있음

즉, 대부분의 프로그램에서 코드 문장을 반복해야하는 경우가 많음<br>
C++은 코드를 반복할 때 사용하는 구문으로 while문, for문, do-while문을 제공함

### 후위 표현식과 전위 표현식 

반복문을 구성할 때는 반복 횟수를 확인하고 적절한 시점에 종료할 수 있게, Counter를 만들어야함 <br> 
후위증가, 전위증가식을 제공<br>
lvalue++ <br>
lvalue-- <br>
++lvalue <br>
--lvalue <br>

**차이**

부가 작용 (변수의 값)에만 주목하면 둘 다 완전히 같은 결과가 나옴 <br>
차이점은 리턴값인데, 후위 표현식은 원래 값을 리턴하며, 전위 표현식은 부가 작용이 일어난 뒤에 값을 리턴함 <br>
그렇기 때문에, counter로 사용할 때는 두 표현식에 차이가 없다고 생각하고 써도 됨.<br>
일반저그올 대부분은 후위표현식을 사용함 

#### 반복문의 종류 
C++에서의 반복문은  while문, for문, do-while문로 3가지가 있음<br>
while, for 반복문 : 조건을 먼저 확인<br>
do-while 반복문 : 반복문 조건을 나중에 확인. 

---------------------
### 5.2 while 반복문 
while 반복문은 조건을 반복하여 평가하고, 참이면 문장 실행, 거짓이면 반복을 종료후 반복문 이후의 문장들을 실행. <br>
while(조건식){

문장;

}

while문의 조건이 항상 참이면 무한 반복함으로, 조건을 거짓으로 만들어서 반복문을 벗어날 수 있도록 해야함<br>
이를위해 카운터 제어와 이벤트 제어를 활용 

+ **카운터 제어**

반복을 몇 번 해야 하는지 알고 있을 때 사용 <br>
반복문에 진입하기 전에 카운터를 초기화함. <br>
그 다음 반복문 내에서 이 값을 증가 또는 감소를 시킨 뒤, 조건을 활용해서 반복을 종료하게 만드는 방법  <br>

+ **이벤트 제어**

  여러 번 반복하는데, 어떤 상황에서 멈춰야 하는지만, 알고 몇 번 반복해야 할지 모르는 경우 <br>

+ Sentinel 제어 while 반복문  <br>
권한이 없는 사람이 특정한 곳에 들어가지 못하게 함  <br>
데이터처리에서 센티넬은 처리 중지를 나타내기 위해 데이터 리스트에 존재하는 특별한 항목임.  <br>
일반적으로 센티넬은 리스트의 다른 항목과 같은 자료형을 갖지만, 값은 특별한 의미를 갖고 있어야 함.  <br>
즉, while 반복문의 조건에 센티넬을 활용하면, 센티넬을 만나는 순간 반복문이 종료 됨  <br>

+ EOF 제어 while 반복문  <br>
키보드와 파일로부터 데이터를 읽어 들일 때, "더 이상 키보드로부터 입력된 데이타가 없다"와 "파일의 끝부분에 도달했습니다" ㄹ하는것을 알리기 위한 센티넬,  End of File라고 함  <br>
예를 들어 유닉스에서는 ctrl+d, 윈도우 에서는 ctrl + z임  <br>
파일을 데이터 소스로 활용할 때는 별도의  처리가 필요 없지만, 파일을 생성할 때는 자동으로 EOF 맠가 추가 됨. <br>
EOF를 사용하려면 ">>" 연산자를 입력. <br>
">>" 연산자는 스트림에서 항목을 읽다가 eof를 만나면 EOF를 읽어 들이지 않고 단순히 FALSE를 리턴함 (반복문 종료) <br>
EOF가 아닌 항목을 만나면 부가 작용을 일으키고 TRUE를 리턴함 (반복문 실행)  <br>

+ 플래그 제어 while 반복문 <br>
센티넬은 리스트의 마지막 항복에 배치 됨. 하지만 중간에 반복문을 종료하고 싶을 수 도 있음  <br>
플래그는 불 변수의 다른 표현 <br>
반복문에 들어가기 전에 플래그를 false로 설정하고, 반복문 내부에서 어떤 조건이 발생할 때, 이를 true로 변경하면서 조건을 활용 <br>
하지만 조건이 발생하지 않을 경우를 대비해서 다른 제어 매커니즘과 함께 사용해야함.  <br>
예를 들어 플래그 제어 + 센티널 조건을 결합하는 상태로 사용  <br>

#### while 반복문 사용시 주의 사항

1. 반복하지 않은 반복문 <br>
2. 무한하게 반복하는 반복문 <br>
3. 본문이 없는 반복문 <br>
4. 조건으로 부동 소수점을 사용하는 반복문

#### 지연 발생
C++에서는 일시 정지하기 위해 지연 기능을 제공함 <br>
while반복문을 사용해서 지연을 만들 수 있음.

### 5.3 for 반복문 

+ **for 반복문 구조**
카운터 제어 반복문을 더 쉽게 사용할 수 있도록 for 반복문을 제공 <br>
for문은 반복 전 초기화, 조건, 변경이라는 3가지가 결합된 헤더를 가짐 <br>

for(초기회; 조건; 변경){

실행할_문장;

}

+ **초기화**

초기화 부분은 카운터를 초기화하는 표현식<br>
일반적으로 카운터에 i, j, k같은 이림의 변수를 사용 <br>
카운터 변수는 반복문 헤더 앞에서 선언할 수도 있고, ㅎ ㅔ더 내부에서 초기화하면서 선언할 수도 있음.

+ **조건**

헤더의 두 번째 부분, while 반복문의 조건처럼 불 표현식을 사용 <br>
이 조건이 참일 때 반복문의 본문을 실행함 <br>
for문도 while문 처럼 본문을 실행하기 전에 조건을 먼저 비교 <br>
만약 조건 부분이 비어 있으면 true가 입력된 것으로 간주 함 

for(int i = 0; i++) { ... } <br>
조건 부분이 아예 없으므로, 컴파일이 되지 않음. 

for(int i = 0; ; i++) { ... }<br>
조건 부분이 비어있으므로, true로 간주되어 무한하게 반복이 일어남.


+ **변경**
  반복문의 본문이 실행된 이후에 변경 부분을 실행함. <br>
  변경 후에는 조건을 다시 확인해서 본문을 실행할지 안 할지를 결정 <br>
  반복문의 본문 뒤에서 변경 작업을 하는 경우, 변경 부분에서 아무 것도 하지 않아도 됨.

#### for문의 본문 
for문은 카운터를 변경하는 부분이 헤더에 들어감 <br>

### 5.4 do-while 반복문

+ **do-while 반복문 구조**
  조건 확인이 본문을 실행 한뒤에 일어남 <br>
  즉, 최소한(적어도) 1번은 실행 됨

  do {

문장 

}
while(조건)**;** 

  

  do-while문은 반드시 뒤에 **세미콜론**이 있어야함 <br>

  + **적어도 한 번 반복** <br> 
    do-while문은 조건과 관계 없이 반복문의 본문을 적어도 한 번 이상 실행해야 할 경우에 사용

  + **데이터 유효성 검사** <br>
  적어도 한 번 이상의 반복이 ㅍ릴요한 또 다른 예로 데이터 유효성 검사(data validation)이 있음 <br>
  사용자로부터 어떤 기준에 맞는 데이터를 입력 받아야하는 프로그램에는 사용자가 올바른 형태로 데이터를 입력했는지 확인하는 데이터 유효성 검사가 있어야 함 <br>
   예를 들어, 어떤 사이트에 가입할 때 이메일, 전화번호 등의 필 수 입력 항목을 확인 하는 것이 데이터 유효성 검사임.

+ **do-while문 분석**
  while 문의 조건 확인 횟수는 반복 횟수보다 1만큼 많음 <br>
  이는 while문이 반복문의 본문을 실행하기 전에 조건을 확인하기 때문에 일어나는 현상임 <br>
  for문도 while문과 비슷한 형태이므로 조건 확인 횟수가 반복 횟수보다 1만큼 많음 <br>
  하지만 do-while문은 조건을 확인하기 전에 본문을 실행하기 때문에, 조건 확인 횟수와 반복 횟수가 같음

  ### 5.5 중첩 반복문

   + **while, for, do-while문 비교**

  while문과 for문은 본문 실행 전에 조건을 확인 함. 하지만 do-while문은 본문을 실행한 후에 조건을 확인 <br>
  while문과 do-while문은 일반적으로 이벤트 제어 방식으로 사용, 반면 for문은 카운터 제어 방식으로 사용 <br>
  while문과 for문의 최소 반복 횟수는 0임. 하지만 do-while문은 적어도 한 번 이상은 실행됨 <br>
  n번의 반복이 일어날 경우, while문과 for문은 n+1번 조건을 확인, 반면에 do-while문은 n번 조건을 확인 <br>

  + **중첩 반복문**
    반복문 내부에 반복문을 넣어 구성하는 것 <br>
    외부의 반복문과 내부의 반복문이 서로 다른 반복문 조합으로도 사용할 수 있음 <br>

  ### 5.6 return, break, continue, goto문

  + **return문**
    return 구문은 현재 함수를 종료하고, 함수를 호출한 쪽으로 제어를 리턴 <br>
    반복문에서 리턴을 사용하면 반복문이 종료되며 동시에 반복문이 소속된 함수도 종료됨 <br>

    [5-17_Program 코드 참고](https://github.com/ysungJ/Cplusplus-UE/blob/main/CPP/Forouzan%20Cpp%20Bible/F_Chapter5/5-17_Program.cpp) <br>
    소수는 자기 자신과 1로만 나눌수 있는 수고, 복합수는 소수와 1이 아닌 수임<br>
    CS에서 소수는 중요하게 사용됨 <br>

    어떤 숫자가 1이라면 소수가 아님 <br>
    어떤 숫자가 1보다 크고 어떤 숫자보다 작은 수로 나누어지지 않은다면 소수임 <br>

    1부터 어떤 숫자까지 모두 나누어 보며 확인할 필요는 없고, 1을 제외한 단 하나의 숫자로라도 나누어 진다면, 해당 숫자는 소수가 아님 <br>

    + **break문**
    break문은 반복문 또는 switch 조건문에서 사용할 수 있는 구문임. <br>
반복문 내부에서 break문을 사용하면 반복을 중간에 종료하고, 반복문을 벗어남 <br>
하지만 break문을 사용하는 것을 추천하지 않으며, 대신 플래그를 사용하는 것을 추천함.

    + **continue문**
    break문은 반복문을 종료하고 벗어남. 하지만 현재 반복만 벗어나고, 반복문 전체를 종료하고 싶지 않을 때, continue문을 사용함  <br>
    다만 while문과, do-while문을 사용할 때는 continue문에 주의해야 함 <br>
while문과 do-while문에서 조건을 변경할 수 있는 문장을 실행하기 전에 continue문이 오면 무한 반복에 빠짐 <br>
반면 for문의 경우 변경 부분이 별도로 있으므로 신경 안써도 됨<br>

-----------------
while(표현식){

if(조건) 

continue;

}

-----------------
do{

...

if(조건) 
  
  continue;

...

}

-------------------
for(초기회; 표현식; 변경){

 ....

 if(다른조건식)

 continue;

#### **goto문**

goto 레이블 

레이블은 동일한 함수 내부에서 레이블 구문이 표시된 부분을 의미함 <br>
 대부분은 goto 구문을 사용하면 프로그램의 구조가 깨지고, 코드가 스파게티 코드가 되어,
  코드를 이해하거나 디버그 하기 힘들다는 이유로 goto문을 잘 사용하지 않음.
